System Prompt: Build a Full-Stack Serverless Anime Streaming Web Application
Tech Stack: Next.js (App Router, Node.js runtime), Vercel, Tailwind CSS, Supabase (PostgreSQL), and @consumet/extensions.

---

### PROJECT OVERVIEW
You are an expert full-stack developer. Build a production-ready, mobile-responsive anime streaming web application. The application will pull video streams dynamically via open-source scrapers, allow users to synchronize their watch history directly with MyAnimeList (MAL) via OAuth2, and host a real-time global comment system under every episode via Supabase.

---

### 1. FILE STRUCTURE & ARCHITECTURE
Generate a Next.js project with the following file arrangement:
- /src/app/page.tsx (Landing page with trending/recent anime)
- /src/app/anime/[id]/page.tsx (Anime details page with episode selection grid)
- /src/app/watch/[episodeId]/page.tsx (Video player client component with integrated Supabase comment section underneath)
- /src/app/api/anime/search/route.ts (API wrapper for Consumet searching)
- /src/app/api/anime/stream/route.ts (Serverless API to extract raw M3U8 video sources)
- /src/app/api/auth/mal/route.ts (MAL OAuth initiation URL generation)
- /src/app/api/auth/callback/route.ts (MAL OAuth code-to-token exchanger)
- /src/app/api/mal/sync/route.ts (Updates MAL user watch status)

---

### 2. CORE FEATURE SPECIFICATIONS

#### STEP A: STREAMS & DATA FETCHING (@consumet/extensions)
1. Initialize the `@consumet/extensions` package inside the `/api/anime/...` backend routes using the `ANIME.Gogoanime` provider.
2. In `/api/anime/stream/route.ts`, accept an `episodeId` search parameter. Use `gogoanime.fetchEpisodeSources(episodeId)` to fetch video fragments. Return the final stream configuration object containing headers, `sources`, and `download` links as clean JSON.

#### STEP B: FRONTEND CHROME & STREAMING PLAYER
1. Create a clean, dark-themed UI reminiscent of modern streaming UI configurations using Tailwind CSS (`bg-zinc-950`, `text-white`).
2. Build an immersive client-side media player component inside `/watch/[episodeId]/page.tsx` utilizing `video.js` or `hls.js`. 
3. Crucial Requirement: Ensure the custom video player configuration maps the HTTP `Referer` headers received from the Consumet streaming API payload to bypass 403 Forbidden cloud restrictions on the stream provider servers.

#### STEP C: MYANIMELIST OAUTH2 TRACKING INTEGRATION
1. Implement the formal MAL OAuth2 Code Challenge/PKCE workflow securely.
2. Inside `/api/auth/mal/route.ts`, generate a cryptographically secure `code_challenge` string, save it server-side or via an encrypted HTTP-Only cookie, and redirect the user to the MAL Authorization URL with your Client ID.
3. In `/api/auth/callback/route.ts`, handle the inbound authorization query parameter `code`. Issue a POST request to `https://myanimelist.net` to securely exchange the code for an authorization payload containing a `refresh_token` and an `access_token`. Commit these variables securely to client session storage or safe encrypted cookies.
4. Implement a background hook: when a video progress indicator crosses 85% completion on the UI layer, launch a fetch operation hitting `/api/mal/sync` to trigger a `PUT` transaction targeting `https://myanimelist.net{anime_id}/my_list_status` updating `num_watched_episodes`.

#### STEP D: SUPABASE REAL-TIME COMMENT ENGINE
1. Incorporate `@supabase/supabase-js` inside your application architecture.
2. Scaffold a clean comments visual interface component positioned directly underneath the media viewport in the watch interface.
3. Implement a fetch routine pulling all commentary historical data bound specifically to the current `episodeId` key.
4. Implement an input form block allowing visitors to register comments. Write a mutation endpoint to `insert` data objects directly to your schema database. Include inputs: `id` (Auto-increment), `episode_id` (Text), `user_name` (Text), `comment_text` (Text), and `created_at` (Timestamp).
